# Distributed Systems Patterns

> General repeatable solutions to commonly occurring system design problems

- Common solutions to problems that involve `multiple components` running as `separate runtime instances`

- **[decomposition/](decomposition/)** — how to split a system into deployable units (multi-tier, microservices, strangler fig)
- **[communication/](communication/)** — how services exchange information (event-driven)
- **[data-management/](data-management/)** — consistency and reliability across services (saga, event sourcing, CQRS, transactional outbox, idempotency)
